Call Logging under PlayStation 2

Discussion

Miles contains a powerful diagnostic feature that allows developers to create a list of all of their application's foreground Miles calls, along with their parameters and results. The call log will be sent to an application callback routine registered with AIL_register_trace_callback, or to the PS2 debugging console via scePrintf if no debug-trace callback function has been registered by the application.

To take advantage of MSS's built-in logging without recompiling your application, you can create a text file called mss_debug_log.txt in the root directory of your PS2 development host's hard drive (d:\ from a PC). If this file exists, then during execution of all subsequent Miles calls, a timestamped API call log will be passed to scePrintf.
