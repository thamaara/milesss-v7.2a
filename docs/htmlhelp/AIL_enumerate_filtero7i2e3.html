Miles Sound System SDK 7.2a

AIL_enumerate_filter_properties

The function AIL_enumerate_filter_properties enumerates a filter's provider-level properties.

S32 AIL_enumerate_filter_properties(
  HPROVIDER filter,
  HINTENUM *next,
  RIB_INTERFACE_ENTRY *dest
);

In Parameters

filter
specifies the filter whose provider-level properties should be enumerated.

Out Parameters

next
is a pointer that MSS uses to keep track of the next property to return. You initialize the next variable with the constant HINTENUM_FIRST before the first call to AIL_enumerate_filter_properties.
dest
points to an structure of type RIB_INTERFACE_ENTRY that receives the property information. The most important field in this RIB structure is "entry_name" which is a pointer to the property name (which can then be used in the AIL_filter_property function.

Returns

0 if no further filter properties were found; non-zero if another property was found.

Discussion

Provider-level properties are properties that are set globally for a particular filter module. For properties that are specific to each sample (sample-level properties), use the AIL_enumerate_sample_stage_properties function. For properties that are specific to an output filter's binding to its HDIGDRIVER, use AIL_enumerate_output_filter_driver_properties.

Example files that use this function: filter.cpp, multiflt.cpp
