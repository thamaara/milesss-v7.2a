Miles Sound System SDK 7.2a

AIL_set_room_type

The function AIL_set_room_type sets the room type that the listener is currently in.

void AIL_set_room_type(
  HDIGDRIVER driver,
  S32 room_type
);

In Parameters

driver
indicates the driver representing the listener.
room_type
indicates the one of the MILES_ROOM_TYPE constants.

Discussion

Use ENVIRONMENT_GENERIC to use the provider's default settings. Use AIL_set_sample_reverb_levels to control the amount of room effects applied to each sample.

It's mandatory to use AIL_set_sample_reverb_levels to turn the wet effects level up for any 3D samples that should have reverb. The per-sample wet level defaults to 0.0, so if you're not hearing your selected room effects, this is probably why!

You can find out if the provider you are using supports room types by querying the current room type with AIL_room_type. If it returns a -1, then room-type control is unsupported.

This function will have no effect with per-voice hardware drivers that don't support reverb in hardware. Currently, this applies only to the MSS_MC_DIRECTSOUND3D value from the MSS_MC_SPEC list, since the EAX providers handle reverb directly and the software-mixing options emulate EAX.

Example files that use this function: examms.cpp
