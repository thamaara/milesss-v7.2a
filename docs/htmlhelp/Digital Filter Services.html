Miles Sound System SDK 7.2a

Digital Filter Services

Discussion

Filter processors are external plug-ins that can manipulate digital sample data. Common filter processors include equalizers, low or high-pass filters, and chorus/flanging filters. Under Windows, filter processors themselves are simply Win32 or Win64 DLLs that have the "FLT" file extension. Under MacOS, filter processors are shared libraries. At startup time, MSS loads any filters found in the same directory as the MSS32.DLL file (or a directory specified by AIL_set_redist_directory).

On platforms that can't use Win32 DLLs, your application must enable filter support by calling Register_RIB(DSP) prior to calling AIL_startup. Note that software DSP filters may be unsuitable for performance-critical applications on the original Xbox. The original Xbox has specific dedicated DSP hardware that you may wish to use instead. See the Xbox docs for details.

Miles currently includes thirteen different external filter modules. They include: capture, high-pass, band-pass, parametric equalizer, shelving equalizer, delay, resonator, phaser, chorus, flange, ring/amplitude modulator, compressor, and a volume ramper. Miles also includes low-pass and high-quality reverb filtering built right into our software mixer.

As with all of Miles, we include full source code for all of the filters (see the \src\sdk directory).

By default, filter processors are not installed on any digital driver. To install a filter processor to be the default filter processor for a digital driver, use the AIL_set_digital_driver_processor function. To install a filter processor for a specific sample (which is by far the more-common application), use the AIL_set_sample_processor function. Each individual sample can have up to eight sample processors installed at one time. Filters and samples can be mixed and matched at will -- you could have a compressor filter on one sample and a low-pass filter on another, for instance.

Filter processors can contain both provide-level and sample-level settings. A provider-level setting might be the amount of memory it should use. A sample-level setting might be the mix level to use for a given HSAMPLE. As with all RIB-based plug-ins, you read and change the settings through the use of "properties".

The MULTIFLT.CPP and FILTER.CPP example programs are a great way to come up to speed on MSS's audio-filtering capabilities.

Digital Filter Services
AIL_set_redist_directory sets the location of your application's Miles directory.
AIL_startup initializes the Miles Sound System and prepares for the installation and use of sound drivers.
AIL_find_filter tries to locate a filter by its name.
AIL_enumerate_filters enumerates the available filter processors for display or for opening.
AIL_open_filter opens a filter to prepare it for use.
AIL_close_filter closes a previously opened filter.
AIL_set_sample_processor assigns the filter or mixer stages for a specific sample.
AIL_sample_processor returns the current filter or mixer stage provider for a specific sample.
AIL_digital_driver_processor returns the current default filter or mixer stage provider for a specific driver.
AIL_digital_output_filter returns the output filter (either per-voice or matrix) provider for a specific driver.
SAMPLESTAGE describes the stage in the sample pipeline to install the filter on.
AIL_set_digital_driver_processor assigns the default filter or mixer stages for a digital driver.
DIGDRVSTAGE describes the stage in the driver pipeline to install the filter on.
AIL_enumerate_filter_properties enumerates a filter's provider-level properties.
RIB_INTERFACE_ENTRY specifies a structure that contains RIB property information.
RIB_ENTRY_TYPE describes the type of the RIB entry.
RIB_DATA_SUBTYPE describes the variable type of the RIB entry.
AIL_enumerate_filter_sample_properties enumerates the possible sample-level filter properties.
AIL_enumerate_sample_stage_properties enumerates the possible sample-level filter properties for the filter currently assigned to a given HSAMPLE processing stage.
AIL_filter_property retrieves and/or sets the current value of the specified provider-level property.
AIL_sample_stage_property retrieves and/or sets the current value of the specified sample-level property.
