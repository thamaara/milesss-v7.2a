Miles Sound System SDK 7.2a

Delay Filter

Discussion

This filter provides a simple delay effect.

Name for use with AIL_find_filter: "Delay Filter"

Properties:

"Max Delay Time": Forces immediate reallocation of internal buffers to support delay times up to the given maximum. Range: 0.01 to 10000.0 milliseconds.

"Delay TimeL": Delay time for left channel. Range: 0.01 to "Max Delay Time" milliseconds.

"Delay TimeR": Delay time for right channel. Range: 0.01 to "Max Delay Time" milliseconds.

"Delay Feedback": Feedback regeneration of signal. Range: 0.0 to 1.0.

"Delay Feedforward": Feedforward gain. Range: 0.0 to 1.0.

"Mix": Mix of delayed signal. Range: 0.0 (original - dry) to 1.0 (full delay - wet).

When a mix value of 0.0 is specified, the filter will operate on an internal "fast path" designed to consume minimal CPU resources.

By default, the "Max Delay Time" property value is set to 300 milliseconds when the filter is initially bound to a sample. This allows you to set "Delay TimeL" and "Delay TimeR" values from 10 milliseconds to 300 milliseconds. Values outside of this range will be clamped; the only way to obtain longer delays is to set a new "Max Delay Time" property value.

Because changes to "Max Delay Time" always cause internal memory buffers to be freed and reallocated, you must not change "Max Delay Time" from within a callback function. The other preferences may be changed at any time.

Per-sample memory requirements for the internal delay buffers may be determined as follows: 

   (8 * "Max Delay Time" in milliseconds * driver output rate in Hz) / 1000 bytes
